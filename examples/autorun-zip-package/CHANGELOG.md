# Changelog

All notable changes to the autorun-zip-package example will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2024-12-09

### Added

-   Initial release of autorun-zip-package example
-   Complete version-controlled autorun.zip package structure
-   `autozip.brs` - BrightSign OS 7.0.60+ extraction script
-   `autorun.brs` - Sample application with HTML widget
-   `index.html` - Sample HTML UI with modern gradient design
-   Sample directory structure (config, logs, brightsign-dumps)
-   `app-config.json` - Example configuration file
-   Cross-platform Node.js build script
-   Comprehensive README with Partner Gallery integration guide
-   STRUCTURE.md documenting correct zip structure
-   Package.json for npm/yarn integration
-   .gitignore for build artifacts

### Documentation

-   Partner Gallery submission requirements
-   Testing procedures (SD card and App URL methods)
-   Troubleshooting guide
-   Hosting recommendations for production
-   Integration with BrightSign provisioning protocol

### Features

-   Cross-platform build support (macOS, Linux, Windows)
-   Preserves empty directories with .gitkeep files
-   Automatic validation of required files
-   Detailed build output and package contents listing
-   Color-coded console output for better UX

### References

-   Implements official BrightSign autorun.zip specification
-   Compatible with BrightSign Partner Gallery requirements
-   Follows dev-cookbook standards and best practices
